# 源代码及部署说明 (SDP)

**文档版本**: v1.0.0
**状态**: Release
**更新日期**: 2026-04-10

## 1. 源代码清单与依赖

### 1.1 目录结构
*   `host-app/`: Electron 上位机源码 (HTML/CSS/JS)。
*   `slave-sim/`: Python 模拟器源码 (`simulator.py`, `render_engine.py`)。

### 1.2 核心依赖清单
*   **后端 (Python 3.12+)**: `pygame>=2.5.0`, `PyOpenGL>=3.1.0`, `opencv-python>=4.8.0`, `numpy>=1.24.0`。
*   **前端 (Node.js 18+)**: `electron ^28.0.0`, `electron-builder ^24.9.1`。

## 2. 编译打包脚本

### 2.1 后端打包 (PyInstaller)
由于目标环境可能无 Python，使用 PyInstaller 打包为独立 EXE。
*   **构建脚本 (`slave-sim/build.bat`)**:
    ```bat
    python -m pip install -r requirements.txt
    python -m pip install pyinstaller
    python -m PyInstaller --noconfirm --onefile --windowed --name "RobotSimulator" --add-data "assets;assets" simulator.py
    ```
*   **产物**：`slave-sim/dist/RobotSimulator.exe`。

### 2.2 前端打包 (Electron Builder)
为避免无代码签名证书导致的安装拦截，采用 `dir` 模式输出绿色免安装包。
*   **构建配置 (`host-app/package.json`)**:
    ```json
    "win": {
      "target": ["dir"],
      "verifyUpdateCodeSignature": false,
      "forceCodeSigning": false
    }
    ```
*   **构建脚本 (`build_host.bat`)**:
    ```bat
    cd host-app
    set ELECTRON_BUILDER_ALLOW_UNRESOLVED_DEPENDENCIES=true
    npm install && npm run build
    ```
*   **产物**：`host-app/dist/win-unpacked/` 目录。

## 3. 部署与回滚策略
*   **部署步骤**：
    1. 将 `RobotSimulator.exe` 与 `win-unpacked` 文件夹打包发送至目标机。
    2. 确保 `RobotSimulator.exe` 同级或内部包含 `assets/real_sewer.jpg`。
    3. 先双击启动模拟器，再启动 `Pipeline Robot Control.exe`。
*   **回滚方案**：由于系统为完全本地化的绿色免安装架构，回滚只需直接删除当前文件夹，解压上一个版本的 ZIP 包即可，实现秒级回滚。

# 软件部署说明 (SDP)
**文档版本**: v1.0.0
**状态**: Release

*说明：由于本项目为【桌面端控制软件 + 本地模拟器】架构，并非云原生 Web 服务，因此部署策略从 K8s/Helm Chart 适配为“跨平台桌面端自动打包与发布”。*

## 1. 部署架构 (Deployment Nodes)
*   **Host Environment**: Windows 10/11 x64 (操作员工作站)
*   **Artifacts (交付物)**:
    1.  `Pipeline Robot Control.exe` (绿色免安装版 / NSIS 安装包)
    2.  `RobotSimulator.exe` (Python 独立可执行文件)
    3.  `assets/real_sewer.jpg` (运行时动态加载的贴图)

## 2. 构建与打包配置 (IaC 替代方案)

### 2.1 模拟器构建 (PyInstaller)
采用 `pyinstaller` 将 Python 脚本及 OpenGL/OpenCV 依赖打包为独立 exe。
*   **脚本 (`slave-sim/build.bat`)**:
    ```bat
    python -m pip install -r requirements.txt
    python -m pip install pyinstaller
    python -m PyInstaller --noconfirm --onefile --windowed --name "RobotSimulator" --add-data "assets;assets" simulator.py
    ```

### 2.2 上位机构建 (Electron Builder)
采用 `electron-builder` 构建。在 `host-app/package.json` 中配置：
```json
"build": {
  "appId": "com.robotsoft.hostapp",
  "productName": "Pipeline Robot Control",
  "win": {
    "target": ["dir"], 
    "icon": "icon.ico",
    "verifyUpdateCodeSignature": false
  },
  "directories": { "output": "dist" }
}
```
*注：采用 `dir` 模式输出绿色免安装包，避免客户现场缺少签名证书导致的安装拦截。*

## 3. CI/CD 流水线 (GitHub Actions)
为实现自动化发布，在 `.github/workflows/release.yml` 中定义如下流水线：

```yaml
name: Build and Release
on:
  push:
    tags:
      - 'v*'

jobs:
  build-simulator:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with: { python-version: '3.12' }
      - name: Build EXE
        run: |
          cd slave-sim
          pip install -r requirements.txt
          pip install pyinstaller
          pyinstaller --onefile --windowed --name "RobotSimulator" --add-data "assets;assets" simulator.py
      - uses: actions/upload-artifact@v3
        with:
          name: Simulator-Windows
          path: slave-sim/dist/RobotSimulator.exe

  build-host:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v3
      - name: Use Node.js
        uses: actions/setup-node@v3
        with: { node-version: '18' }
      - name: Build Electron App
        run: |
          cd host-app
          npm install
          npm run build
      - uses: actions/upload-artifact@v3
        with:
          name: HostApp-Windows
          path: host-app/dist/win-unpacked/
```

## 4. 环境配置与日志监控
*   **配置文件**: 无需远程配置中心。模拟器启动后，会在运行目录下自动创建 `outputs/` 文件夹用于存放录像与截图。
*   **日志**: 
    *   **前端日志**: 直接输出至 Electron 界面的 Console Panel（彩色格式化）。
    *   **后端日志**: `RobotSimulator.exe` 若移除 `--windowed` 参数，可在控制台实时查看 TCP/UDP 通信状态。
