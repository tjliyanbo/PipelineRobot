# 需求追踪矩阵 (Requirements Traceability Matrix - RTM)

| 需求ID | 原始需求描述 (来源：合同/用户提示) | 功能模块 | 实现策略 (Host) | 实现策略 (Simulator) | 验证方法 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **REQ-001** | **双链路通信**：视频与控制指令分离，防止互相阻塞 | 驱动层/通信模块 | Node.js `net` 模块处理 TCP 控制，`dgram` 模块处理 UDP 视频接收 | Python `asyncio` 处理 TCP，`socket` 处理 UDP 推流 | 断开 TCP 连接，验证视频流是否持续 |
| **REQ-002** | **高保真3D渲染**：管道内壁、物理照明、PBR材质模拟 | 模拟器/3D渲染 | N/A | 使用 `PyOpenGL` 渲染圆柱体，实现双面光照、高光反射贴图 | 视觉走查：检查光影与纹理真实度 |
| **REQ-003** | **真实照片映射**：支持将真实下水道照片无缝贴合至3D管道 | 模拟器/纹理处理 | 发送切换模式指令 (`0x11`) | 使用 OpenCV `cv2.warpPolar` 进行透视展开，并作为 OpenGL 纹理 | 单元测试：检查接缝处是否有拉伸变形 |
| **REQ-004** | **工业级UI设计**：高对比度暗色主题，适配严苛环境 | 表现层/UI框架 | 使用 CSS Variables 定义赛博青/深碳灰主题，设计悬浮视频工具栏 | N/A | WCAG 对比度工具测试 (达 AAA 级) |
| **REQ-005** | **实时视频处理**：保证长宽比，文字叠加清晰，支持录制/截图 | 表现层/视频模块 | 接收 JPEG Base64 渲染，UI 提供 Fill/Fit 比例切换，及录像按钮 | OpenCV 智能 Letterbox 缩放，在下采样后叠加文字，提供 MP4 录制 | 导出录制文件，检查分辨率与水印清晰度 |
| **REQ-006** | **平滑运动控制**：前进/后退/转向需平滑可控 | 业务层/控制逻辑 | 降低发送的速度基数 (0.5)，限制最大输入值 | 降低物理引擎的 Gear Ratio (从 8.0 降至 5.0)，累计平滑位移 | 人工测试：长按/点按方向键，观察画面是否抖动 |
| **REQ-007** | **传感器实时展示**：温度、湿度、气压、电量 | 表现层/仪表盘 | 监听 `telemetry-data` 事件，高频更新 DOM 文本节点 | 每秒 5Hz 频率生成随机漫步传感器数据并通过 TCP 发送 | 性能测试：观察 CPU 占用与界面卡顿 |
| **REQ-008** | **协议安全性**：包含 Header 和 CRC32 校验 | 驱动层/协议解析 | 组包时计算 CRC32，解包时验证头尾 | 接收包时进行 Header (0xAA55) 匹配和 JSON 反序列化 | 异常测试：发送残缺包或错包，观察崩溃情况 |

## 性能指标 (KPI)
*   **端到端延迟**: < 100ms (控制指令)
*   **视频帧率**: ≥ 30 fps (UDP 传输，320x240)
*   **UI 响应**: 纯原生 JS，无框架开销，DOM 渲染 < 16ms
