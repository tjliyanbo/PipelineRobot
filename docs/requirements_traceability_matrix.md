# 需求追踪矩阵 (Requirements Traceability Matrix - RTM)

| 需求ID | 原始需求描述 (来源：合同/用户提示) | 功能模块 | 实现策略 (Host) | 实现策略 (Simulator) | 验证方法 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **REQ-001** | **通信协议**：基于TCP/IP开发，支持双向交互，误码率≤10⁻⁶ | 驱动层/通信模块 | Node.js `net`/`dgram` 模块，封装 Packet 类，实现拆包/粘包处理 | Python `socket` 模块，实现标准协议服务端 | 自动化测试：发送10万包，统计丢包率 |
| **REQ-002** | **心跳与重连**：支持断线重连、心跳检测 (3s超时) | 驱动层/健康监测 | 定时发送 Heartbeat 包，检测 Socket 状态，自动重连机制 | 响应 Heartbeat，检测超时自动断开 | 异常测试：物理断网，观察重连日志 |
| **REQ-003** | **数据校验**：集成CRC校验，确保完整性 | 驱动层/协议解析 | 接收数据计算 CRC32，不匹配则丢弃并记录错误 | 发送数据附加 CRC32 校验码 | 异常测试：注入 Checksum 错误 |
| **REQ-004** | **数据加密**：敏感数据加密存储 (AES-256) | 业务层/安全 | 使用 `crypto` 模块对配置/日志进行 AES-256 加密 | N/A (模拟器只需模拟明文/密文传输协议) | 检查本地存储文件是否乱码 |
| **REQ-005** | **实时视频**：视频流实时传输，支持录制/截图 | 表现层/视频模块 | 集成 WebRTC 或 FFmpeg 播放器，Canvas 截图，文件流录制 | 模拟 RTSP/UDP 视频流推流 (OpenCV/FFmpeg) | 功能测试：目测延迟，检查截图文件 |
| **REQ-006** | **状态显示**：实时刷新传感器数据 (电量/姿态/气压) | 表现层/仪表盘 | React 组件 (Recharts/ECharts) 实时渲染，刷新率≥20Hz | 生成符合正态分布的随机/脚本化传感器数据 | 性能测试：观察 CPU 占用与界面卡顿 |
| **REQ-007** | **运动控制**：全局/精细控制，速度滑块，延迟<100ms | 业务层/控制逻辑 | 键盘/UI 事件映射为控制指令，高优先级队列发送 | 解析控制指令，更新模拟位置/速度状态 | 性能测试：抓包分析指令RTT |
| **REQ-008** | **日志审计**：操作审计日志完整不可篡改 | 业务层/日志 | 记录所有用户操作，计算日志链 Hash | N/A | 检查日志文件完整性 |
| **REQ-009** | **场景脚本**：支持场景脚本批量运行、性能采样 | 模拟器/脚本引擎 | N/A | Python 脚本加载器，解析 JSON/XML 场景文件 | 运行测试脚本，生成报告 |
| **REQ-010** | **异常模拟**：支持通信中断、协议错位等异常注入 | 模拟器/故障注入 | N/A | 增加干扰层 (Middleware)，随机丢包/篡改数据 | 配合 Host 进行异常测试 |

## 性能指标 (KPI)
*   **端到端延迟**: < 100ms (控制指令)
*   **界面响应**: ≤ 300ms
*   **恢复时间**: ≤ 3s (异常后)
